คำถามที่พบบ่อย (FAQ)

❓ ต้องรู้เขียนโค้ดก่อนเรียนคู่มือนี้ไหม?

ไม่จำเป็น! คู่มือนี้ออกแบบสำหรับผู้เริ่มต้น อธิบายทุกขั้นตอน แต่ถ้ามีพื้นฐาน HTML/CSS/JavaScript มาก่อนจะเข้าใจเร็วขึ้น

❓ ใช้ AI เขียนโค้ดทั้งหมดได้ไหม?

ได้ แต่ต้องอ่านโค้ดก่อนใช้เสมอ ตรวจ Security (RLS, Env Vars) ให้ดี และทดสอบก่อน Deploy อย่าเชื่อ AI 100% โดยไม่ตรวจสอบ

❓ ทุกอย่างฟรีจริงหรือ?

ใช่! GitHub (Free), Vercel (Hobby plan), Supabase (Free tier — 2 โปรเจกต์, 500MB) เพียงพอสำหรับเรียนรู้และโปรเจกต์เล็ก-กลาง

❓ เลือก Next.js ทำไม? ใช้ React ธรรมดาได้ไหม?

ใช้ได้ แต่ Next.js มีข้อดีเพิ่มเติม: routing อัตโนมัติ, Server-Side Rendering, API Routes ในตัว และ Vercel สร้าง Next.js เองจึงรองรับดีที่สุด

❓ Push .env ขึ้น GitHub ไปแล้วทำอย่างไร?

1) เปลี่ยน Key/Secret ทันที (สำคัญที่สุด!) 2) เอาไฟล์ออก: git rm --cached .env 3) เพิ่ม .gitignore 4) อัปเดต Key ใหม่ทุกที่ ดูรายละเอียดในบท 🐙 GitHub

❓ เว็บ Deploy แล้วแต่หน้าขาว?

สาเหตุหลัก: Framework ถูกตรวจจับผิด, Root Directory ผิด, หรือ Build error ที่ไม่ได้ดูใน Logs — ไปที่ Vercel Dashboard → Deployments → อ่าน Build Logs

❓ ผู้ใช้คนหนึ่งเห็นข้อมูลของอีกคน?

🚨 RLS อาจยังไม่ได้เปิด หรือ Policy ตั้งผิด! ตรวจว่า: 1) RLS เปิดทุกตาราง 2) Policy ใช้ auth.uid() = user_id 3) ไม่มี USING (true) ใน SELECT

❓ จะเรียนอะไรต่อ?

หลังจบคู่มือนี้แนะนำ: TypeScript เชิงลึก → React Hooks → Next.js App Router → Tailwind CSS → PostgreSQL → Testing
