Storage (เก็บไฟล์)

🟢 Beginner ⏱️ อ่าน ~8 นาที
📋 Prerequisite
✓ Supabase พื้นฐาน

Supabase Storage ให้เราอัปโหลดไฟล์ (รูปภาพ, เอกสาร) และสร้างลิงก์สำหรับดาวน์โหลด

📦 แนวคิดหลัก

คำศัพท์คืออะไรตัวอย่าง
Bucketโฟลเดอร์หลักสำหรับเก็บไฟล์avatars, documents
Policyกฎว่าใครเข้าถึงไฟล์ไหนได้เฉพาะเจ้าของดูรูปตัวเองได้

🪣 สร้าง Bucket

  1. Supabase Dashboard → Storage
  2. กด New bucket
  3. ตั้งชื่อ เช่น avatars
  4. เลือก Public (ถ้าต้องการให้ดูได้โดยไม่ต้อง Login) หรือ Private
  5. กด Create bucket

📤 อัปโหลดไฟล์

ts📄 ตัวอย่างโค้ด
async function uploadAvatar(file: File, userId: string) {
  // ตั้ง path เป็น userId/filename เพื่อจัดการ Policy ง่าย
  const filePath = `${userId}/${Date.now()}_${file.name}`

  const { error } = await supabase.storage
    .from('avatars')       // ชื่อ Bucket
    .upload(filePath, file, {
      cacheControl: '3600',  // cache 1 ชม.
      upsert: false          // false = ห้ามเขียนทับ
    })

  if (error) {
    console.error('อัปโหลดไม่สำเร็จ:', error.message)
    return null
  }

  // สร้าง URL สาธารณะ
  const { data } = supabase.storage
    .from('avatars')
    .getPublicUrl(filePath)

  return data.publicUrl
}

📥 ดาวน์โหลด / ดูไฟล์

ts📄 ตัวอย่างโค้ด
// Public bucket — ดู URL ได้เลย
const { data } = supabase.storage
  .from('avatars')
  .getPublicUrl('userId/photo.jpg')
// ใช้ data.publicUrl ใน <img src={...} />

// Private bucket — สร้าง URL ชั่วคราว (หมดอายุได้)
const { data: signedData } = await supabase.storage
  .from('private-docs')
  .createSignedUrl('userId/secret.pdf', 3600)  // หมดอายุใน 1 ชม.
// ใช้ signedData?.signedUrl

🛡️ Storage Policy

เหมือน RLS ใน Database — ต้องตั้ง Policy เพื่อกำหนดสิทธิ์:

sql⚡ SQL Editor
-- อนุญาตให้ผู้ใช้อัปโหลดเฉพาะในโฟลเดอร์ของตัวเอง
CREATE POLICY "users upload own folder"
  ON storage.objects FOR INSERT
  WITH CHECK (
    bucket_id = 'avatars'
    AND auth.uid()::text = (storage.foldername(name))[1]
  );

-- อนุญาตให้ทุกคนดูไฟล์ใน public bucket
CREATE POLICY "public read avatars"
  ON storage.objects FOR SELECT
  USING (bucket_id = 'avatars');
💜 จัดโฟลเดอร์ตาม User ID

ตั้ง path เป็น userId/filename จะจัดการ Policy ได้ง่าย — ให้ผู้ใช้แต่ละคนเข้าถึงเฉพาะโฟลเดอร์ของตัวเอง

📤 ตัวอย่าง Upload Component

tsx📄 components/AvatarUpload.tsx
'use client'
import { useState } from 'react'
import { supabase } from '@/lib/supabase'

export default function AvatarUpload({ userId }: { userId: string }) {
  const [uploading, setUploading] = useState(false)

  async function handleUpload(e: React.ChangeEvent<HTMLInputElement>) {
    const file = e.target.files?.[0]
    if (!file) return

    // ตรวจขนาดไฟล์ (จำกัด 2MB)
    if (file.size > 2 * 1024 * 1024) {
      alert('ไฟล์ใหญ่เกิน 2MB!')
      return
    }

    // ตรวจประเภทไฟล์
    if (!file.type.startsWith('image/')) {
      alert('อัปโหลดได้เฉพาะรูปภาพ!')
      return
    }

    setUploading(true)
    const path = `${userId}/${Date.now()}.${file.name.split('.').pop()}`

    const { error } = await supabase.storage
      .from('avatars')
      .upload(path, file)

    if (error) alert('อัปโหลดไม่สำเร็จ: ' + error.message)
    else alert('อัปโหลดสำเร็จ! ✅')

    setUploading(false)
  }

  return (
    <label className="cursor-pointer">
      <span className="text-blue-600 hover:underline">
        {uploading ? '⏳ กำลังอัปโหลด...' : '📷 เปลี่ยนรูปโปรไฟล์'}
      </span>
      <input type="file" accept="image/*"
        onChange={handleUpload} disabled={uploading}
        className="hidden" />
    </label>
  )
}
