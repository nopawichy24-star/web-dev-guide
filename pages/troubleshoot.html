📖 วิธีอ่าน Error Message (สำคัญมาก!)

หลายคนเลิกเขียนโปรแกรมเพราะอ่าน Error ไม่เป็น — จริงๆ แล้ว Error พยายามบอกเราว่าผิดตรงไหน!

Error ที่เห็นแปลว่าแก้อย่างไร
Module not found🔍 หาไฟล์/library ไม่เจอตรวจชื่อ import, ตรวจว่า npm install แล้ว, ตรวจ path
Unexpected token📝 Syntax ผิด — พิมพ์อะไรผิดไปดูบรรทัดที่บอก → หา , ; ) } ที่ขาด หรือเกิน
Permission denied🔒 สิทธิ์ไม่พอตรวจ Token/SSH Key, ตรวจ RLS Policy
Cannot read properties of undefined❌ พยายามอ่านค่าจากตัวแปรที่เป็น undefinedตรวจว่าข้อมูลโหลดเสร็จแล้วหรือยัง, ใช้ ?.
TypeError: X is not a function🚫 เรียกใช้สิ่งที่ไม่ใช่ฟังก์ชันตรวจชื่อฟังก์ชัน, ตรวจ import ถูกไหม
CORS error🌐 เบราว์เซอร์บล็อกเพราะข้ามโดเมนตั้ง CORS headers บน Server หรือใช้ API Route
404 Not Found📄 หน้า/ไฟล์ที่ขอไม่มีอยู่ตรวจ URL, ตรวจชื่อไฟล์, ตรวจ routing
500 Internal Server Error💥 Server มีปัญหาดู Server Logs, ตรวจโค้ดฝั่ง Server
RLS: insufficient privilege🛡️ RLS บล็อกเพราะไม่มี Policyเพิ่ม Policy ที่ถูกต้องสำหรับ operation นั้น
JWT expired⏰ Token หมดอายุLogin ใหม่ หรือใช้ onAuthStateChange refresh

git push ถูก rejectGitHub มี Commit ที่เราไม่มีgit pull --rebase ก่อน push
Build failed บน VercelTypeScript error / import ผิดอ่าน Build Logs → แก้ error → push ใหม่
หน้าเว็บขาวหลัง DeployFramework ถูกตรวจจับผิด / Root Dir ผิดVercel Settings → ตรวจ Framework + Root

📋 ปัญหาแยกตามเครื่องมือ

📦 Git

ปัญหาวิธีแก้
fatal: not a git repositoryยังไม่ได้รัน git init หรืออยู่ผิดโฟลเดอร์ → cd ไปให้ถูก
nothing to commitไม่มีไฟล์เปลี่ยนแปลง หรือ add แล้วยังไม่ได้แก้อะไร
merge conflictเปิดไฟล์ → หาเครื่องหมาย <<< → เลือกโค้ด → ลบเครื่องหมาย → add + commit
โค้ดหายหลัง reset --hardลองใช้ git reflog หา commit เก่า → git reset --hard <hash>

🐙 GitHub

ปัญหาวิธีแก้
Push แล้วโดน rejectgit pull --rebase ก่อน push
Secret ถูก Push ไปแล้วเปลี่ยน Key ทันที! → git rm --cached → เพิ่ม .gitignore
Permission deniedตรวจ Token/SSH → สร้าง Token ใหม่ หรือตั้ง SSH Key

▲ Vercel

ปัญหาวิธีแก้
Build failedอ่าน Build Logs → แก้ Error ที่ระบุ → Push ใหม่
Env var undefinedตรวจชื่อ + prefix NEXT_PUBLIC_ + Redeploy
หน้าขาวตรวจ Framework Preset + Root Directory ใน Settings
404 เมื่อเปิด URL ตรงตรวจ routing ของ Framework (Next.js มักไม่มีปัญหานี้)

⚡ Supabase

ปัญหาวิธีแก้
ข้อมูลไม่ขึ้นตรวจ RLS → Policy อาจบล็อกการอ่าน → เพิ่ม SELECT policy
Insert ไม่ได้ตรวจ RLS INSERT policy → WITH CHECK (auth.uid() = user_id)
เห็นข้อมูลคนอื่นRLS ไม่ได้เปิด หรือ Policy ใช้ USING (true)
Login แล้วไม่ redirectตรวจ Redirect URLs ใน Auth → URL Config
อีเมลยืนยันไม่มาดู Spam folder, รอ 1-2 นาที, ตรวจ SMTP settings
