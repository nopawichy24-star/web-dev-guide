คู่มือ GitHub ฉบับละเอียด

🟢 Beginner ⏱️ อ่าน ~15 นาที
📋 Prerequisite
✓ Git พื้นฐาน
✓ มี GitHub Account

GitHub คือเว็บไซต์ที่เก็บโค้ดของเราไว้บน Cloud เพื่อสำรองข้อมูล ทำงานร่วมกัน และเชื่อมกับ Vercel เพื่อ Deploy เว็บ

📝 สมัครบัญชี GitHub

  1. เข้าเว็บ github.com
  2. กด Sign up
  3. กรอกอีเมล, ตั้งรหัสผ่าน, ตั้งชื่อผู้ใช้ (username)
  4. ยืนยันอีเมล
  5. เลือกแผน Free (ฟรีเพียงพอสำหรับงานส่วนตัว)
💜 แนะนำชื่อ Username

ใช้ชื่อที่เป็นมืออาชีพ เช่น nopawich หรือ nopawich-dev — เพราะ URL โปรเจกต์จะเป็น github.com/ชื่อคุณ/ชื่อโปรเจกต์

📁 สร้าง Repository

  1. กดปุ่ม + มุมบนขวา → New repository
  2. ตั้งชื่อ Repository (เช่น my-first-website)
  3. เลือก Public หรือ Private
  4. ✅ ติ๊ก Add a README file
  5. ✅ ติ๊ก Add .gitignore → เลือก Node
  6. กด Create repository

🆚 Public vs Private

🌍 Public

ทุกคนเห็นโค้ดได้ เหมาะสำหรับ Open Source หรือเว็บ Portfolio

🔒 Private

เฉพาะเราและคนที่เชิญเท่านั้น เหมาะสำหรับโปรเจกต์ลูกค้า หรืองานส่วนตัว

🧩 ฟีเจอร์สำคัญของ GitHub

ฟีเจอร์คืออะไรใช้เมื่อไหร่
📘 READMEไฟล์อธิบายโปรเจกต์ทุกโปรเจกต์ต้องมี!
🙈 .gitignoreรายชื่อไฟล์ที่ Git จะไม่ติดตามป้องกัน Secret, node_modules
📜 Licenseสัญญาอนุญาตใช้งานโปรเจกต์ Open Source
🐛 Issuesแจ้งบั๊กหรือเสนอฟีเจอร์ติดตามงานและปัญหา
🔀 Pull Requestขอรวมโค้ดจาก Branch หนึ่งเข้า Branch หลักทำงานเป็นทีม, Code Review
⚡ Actionsระบบทำงานอัตโนมัติ (CI/CD)ทดสอบ, Build, Deploy อัตโนมัติ
🍴 Forkคัดลอก Repo คนอื่นมาเป็นของเราแก้ไข Open Source
⭐ Starกดดาวให้โปรเจกต์ที่ชอบบุ๊คมาร์คโปรเจกต์น่าสนใจ
🔑 Secretsเก็บค่าลับ (API Key) อย่างปลอดภัยใช้ใน GitHub Actions

🔗 เชื่อม Git ในเครื่องกับ GitHub

มี 2 วิธีหลัก:

วิธีที่ 1: HTTPS (แนะนำสำหรับมือใหม่ ✅)

bash💻 Terminal — ในโฟลเดอร์โปรเจกต์
# 1. เชื่อม Git กับ GitHub Repository
git remote add origin https://github.com/username/my-project.git

# 2. ตั้งชื่อ Branch หลักเป็น main
git branch -M main

# 3. Push โค้ดขึ้น GitHub ครั้งแรก
git push -u origin main
💡 เปลี่ยน username เป็นชื่อ GitHub ของคุณ!

URL จะอยู่ในหน้า Repository บน GitHub → กดปุ่ม Code สีเขียว → คัดลอก HTTPS URL มาวาง

เมื่อ Push ครั้งแรก GitHub จะถาม Authentication — ต้องใช้ Personal Access Token แทนรหัสผ่าน:

📌 สร้าง Personal Access Token

  1. เข้า GitHub → กดรูปโปรไฟล์มุมบนขวา → Settings
  2. เลื่อนลงซ้ายสุด → Developer settings
  3. เลือก Personal access tokens → Tokens (classic)
  4. กด Generate new token (classic)
  5. ตั้งชื่อ (เช่น "my-laptop") ตั้ง Expiration
  6. ติ๊ก repo (อย่างน้อย)
  7. กด Generate token
  8. คัดลอก Token ทันที! จะเห็นแค่ครั้งเดียว
🚨 สำคัญมาก!

Token นี้คือกุญแจเข้าบัญชี GitHub ของคุณ ห้ามแชร์ ห้ามวางในโค้ด ห้าม Push ขึ้น GitHub! เก็บไว้ที่ปลอดภัย (เช่น Notes ที่ล็อกรหัส)

วิธีที่ 2: SSH (สะดวกกว่าเมื่อตั้งค่าแล้ว)

bash💻 Terminal
# 1. สร้าง SSH Key (กด Enter ไปเรื่อยๆ ไม่ต้องตั้งรหัส)
ssh-keygen -t ed25519 -C "your@email.com"

# 2. คัดลอก Public Key
# Mac:
cat ~/.ssh/id_ed25519.pub | pbcopy
# Windows (PowerShell):
Get-Content ~/.ssh/id_ed25519.pub | clip

# 3. ไปวางที่ GitHub → Settings → SSH and GPG keys → New SSH key

# 4. ทดสอบการเชื่อมต่อ
ssh -T git@github.com

ผลลัพธ์ที่ดี: Hi username! You've successfully authenticated

เมื่อใช้ SSH ให้ใช้ URL แบบ SSH แทน HTTPS:

bash💻 Terminal
git remote add origin git@github.com:username/my-project.git

🙈 .gitignore — ไฟล์ที่ Git ต้องเพิกเฉย

ไฟล์ .gitignore บอก Git ว่าอย่าติดตามไฟล์อะไรบ้าง สำคัญมากเพื่อป้องกันข้อมูลลับรั่วไหล

gitignore📄 .gitignore — ไฟล์นี้ต้องอยู่ที่ root ของโปรเจกต์
# Dependencies — ไฟล์ที่ npm ติดตั้ง (ใหญ่มาก ไม่ต้อง Push)
node_modules

# Build output
.next
dist
build

# 🚨 สำคัญ! ไฟล์ที่มีข้อมูลลับ
.env
.env.local
.env.production
.env*.local

# OS files
.DS_Store
Thumbs.db

# IDE
.vscode/settings.json
.idea
🚨 ต้องสร้าง .gitignore ก่อน Commit แรก!

ถ้า Commit ไฟล์ .env ไปแล้ว แม้จะเพิ่มใน .gitignore ทีหลัง Git จะยังติดตามอยู่ ต้องใช้ git rm --cached .env เอาออก

🚀 Push โค้ดขึ้น GitHub ครั้งแรก (ทีละขั้นตอน)

สมมติว่ามีโปรเจกต์ในเครื่องแล้ว และสร้าง Repo เปล่าบน GitHub แล้ว:

bash💻 Terminal — ในโฟลเดอร์โปรเจกต์
# 1. เข้าไปในโฟลเดอร์โปรเจกต์
cd my-project

# 2. เริ่ม Git (ถ้ายังไม่เคย git init)
git init

# 3. ตรวจสอบว่ามี .gitignore แล้ว
cat .gitignore

# 4. เพิ่มไฟล์ทั้งหมด
git add .

# 5. ตรวจสอบว่าไม่มีไฟล์ .env หลุดเข้ามา!
git status

# 6. Commit ครั้งแรก
git commit -m "initial commit"

# 7. ตั้งชื่อ Branch เป็น main
git branch -M main

# 8. เชื่อมกับ GitHub (เปลี่ยน URL เป็นของคุณ)
git remote add origin https://github.com/username/my-project.git

# 9. Push ขึ้น GitHub!
git push -u origin main
🎉 สำเร็จ!

เปิด GitHub ในเบราว์เซอร์ → ไปที่ Repo → จะเห็นโค้ดของเราอยู่บนนั้นแล้ว!

🔧 ปัญหาที่พบบ่อยกับ GitHub

ปัญหาสาเหตุวิธีแก้
Permission denied ยังไม่ได้ตั้ง Authentication หรือ Token หมดอายุ สร้าง Personal Access Token ใหม่ หรือตั้ง SSH Key
Authentication failed ใช้รหัสผ่าน GitHub โดยตรง (GitHub ไม่รับแล้ว) ต้องใช้ Personal Access Token แทน
Repository not found URL ผิด หรือ Repo เป็น Private แต่ไม่มีสิทธิ์ ตรวจสอบ URL ให้ตรง ดูด้วย git remote -v
remote origin already exists เคย add remote แล้ว ใช้ git remote set-url origin URL_ใหม่
rejected (non-fast-forward) GitHub มีโค้ดใหม่ที่เราไม่มี รัน git pull --rebase ก่อนแล้ว Push ใหม่
large file error ไฟล์ใหญ่เกิน 100MB ใส่ไฟล์ใหญ่ใน .gitignore หรือใช้ Git LFS

🚨 เมื่อ Secret รั่วไหลขึ้น GitHub

นี่คือสถานการณ์ฉุกเฉินที่สำคัญมาก — ถ้าเราเผลอ Push ไฟล์ .env, API Key หรือ Password ขึ้น GitHub

🚨 คำเตือนสำคัญ

การลบไฟล์ออกจาก Commit ล่าสุดไม่เพียงพอ! เพราะข้อมูลยังอยู่ใน Git History — ใครก็สามารถเข้าไปดู Commit เก่าได้

🛠 วิธีแก้เมื่อ Secret รั่วแล้ว

ขั้นตอนที่ 1 — เปลี่ยน Secret ทันที! (สำคัญที่สุด)

  • ถ้าเป็น Supabase Key → ไปที่ Supabase Dashboard → สร้าง Key ใหม่
  • ถ้าเป็น GitHub Token → ลบ Token เก่า → สร้างใหม่
  • ถ้าเป็น API Key อื่น → เข้าไปที่บริการนั้น → Rotate/Revoke Key

ขั้นตอนที่ 2 — เพิ่ม .gitignore

bash💻 Terminal
# เอาไฟล์ .env ออกจาก Git tracking (ไม่ลบไฟล์จริง)
git rm --cached .env
git rm --cached .env.local

# เพิ่มใน .gitignore (ถ้ายังไม่มี)
echo ".env" >> .gitignore
echo ".env.local" >> .gitignore

# Commit การแก้ไข
git add .gitignore
git commit -m "fix: remove .env from tracking, add to .gitignore"
git push

ขั้นตอนที่ 3 — อัปเดต Key ใหม่ในทุกที่

  • อัปเดตไฟล์ .env.local ในเครื่อง
  • อัปเดต Environment Variables ใน Vercel Dashboard
  • Redeploy บน Vercel
⚠️ จำไว้เสมอ

Secret ที่ถูก Push ขึ้นอินเทอร์เน็ตแล้ว ให้ถือว่าถูกเปิดเผยแล้วตลอดไป แม้จะลบทิ้งก็ตาม วิธีเดียวที่ปลอดภัยคือเปลี่ยน Secret ใหม่ทันที

ไฟล์ README.md คือหน้าแรกที่คนเห็นเมื่อเปิด Repo ควรมี:
