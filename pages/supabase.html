Supabase คือบริการ Backend-as-a-Service ที่ให้เราได้ฐานข้อมูล, ระบบสมาชิก, เก็บไฟล์ และ API พร้อมใช้ทันที

🧩 องค์ประกอบหลักของ Supabase

ส่วนคืออะไรใช้ทำอะไร
🗄️ Databaseฐานข้อมูล PostgreSQLเก็บข้อมูลทั้งหมด (สมาชิก, โพสต์, สินค้า)
👤 Authระบบสมาชิกสมัคร/เข้าสู่ระบบ ด้วย Email, Google, LINE
📁 Storageเก็บไฟล์อัปโหลดรูปภาพ, เอกสาร
⚡ Realtimeข้อมูลอัปเดตทันทีแชท, แจ้งเตือน, Dashboard สด
🛡️ RLSRow Level Securityกำหนดว่าใครเข้าถึงข้อมูลอะไรได้
🔗 APIAPI อัตโนมัติเรียกใช้ข้อมูลจากโค้ด JavaScript

📝 สมัครและสร้างโปรเจกต์

  1. เข้าเว็บ supabase.com → กด Start your project
  2. สมัครด้วย GitHub (แนะนำ — เชื่อมทีเดียวจบ)
  3. กด New project
  4. ตั้งชื่อ Project (เช่น my-task-app)
  5. ตั้งรหัสผ่าน Database — จดไว้ให้ดี! ใช้แก้ไข Database โดยตรง
  6. เลือก Region → เลือก Northeast Asia (Tokyo) ถ้าผู้ใช้อยู่เอเชีย
  7. กด Create new project → รอ 1-2 นาที

📊 สร้าง Table (ตาราง)

ลองนึกว่า Table คือตาราง Excel แต่ละแผ่น:

คำศัพท์เทียบกับ Excelตัวอย่าง
Tableแผ่นงาน (Sheet)ตาราง users, ตาราง tasks
Rowแถว (แต่ละรายการ)ผู้ใช้คนที่ 1, Task ที่ 1
Columnคอลัมน์ (หัวข้อ)ชื่อ, อีเมล, วันที่สร้าง
Primary Keyลำดับที่ (ไม่ซ้ำกัน)id
Foreign Keyลิงก์ไปอีก Sheetuser_id → users.id

สร้าง Table ผ่าน Dashboard

  1. เข้า Supabase Dashboard → Table Editor
  2. กด Create a new table
  3. ตั้งชื่อ (เช่น tasks)
  4. ✅ เปิด Enable Row Level Security (สำคัญมาก!)
  5. เพิ่ม Column ตามต้องการ
  6. กด Save

หรือสร้างด้วย SQL

ไปที่ SQL Editor แล้วรัน:

sql⚡ Supabase → SQL Editor
-- ตาราง profiles (ข้อมูลผู้ใช้)
CREATE TABLE profiles (
  id UUID PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
  username TEXT UNIQUE,
  full_name TEXT,
  avatar_url TEXT,
  created_at TIMESTAMPTZ DEFAULT now()
);

-- ตาราง tasks (รายการงาน)
CREATE TABLE tasks (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES profiles(id) ON DELETE CASCADE NOT NULL,
  title TEXT NOT NULL,
  completed BOOLEAN DEFAULT false,
  created_at TIMESTAMPTZ DEFAULT now()
);

-- เปิด RLS ทั้ง 2 ตาราง (สำคัญมาก!)
ALTER TABLE profiles ENABLE ROW LEVEL SECURITY;
ALTER TABLE tasks ENABLE ROW LEVEL SECURITY;
⚠️ เปิด RLS ทุกตารางเสมอ!

ถ้าไม่เปิด RLS ใครก็สามารถอ่าน/แก้/ลบข้อมูลทุกอย่างได้ผ่าน anon key ที่อยู่ในโค้ด Frontend! ดูรายละเอียดเรื่อง RLS ในบท 🛡️ Security

📝 SQL พื้นฐานที่ต้องรู้

อ่านข้อมูล

sql⚡ SQL Editor
-- อ่านทุกแถว ทุกคอลัมน์
SELECT * FROM tasks;

-- อ่านเฉพาะบางคอลัมน์
SELECT title, completed FROM tasks;

-- กรองด้วยเงื่อนไข
SELECT * FROM tasks WHERE completed = false;

-- เรียงลำดับ
SELECT * FROM tasks ORDER BY created_at DESC;

-- จำกัดจำนวน
SELECT * FROM tasks LIMIT 10;

-- รวมเงื่อนไข
SELECT * FROM tasks
  WHERE user_id = 'xxx' AND completed = false
  ORDER BY created_at DESC
  LIMIT 20;

เพิ่มข้อมูล

sql⚡ SQL Editor
INSERT INTO tasks (user_id, title)
VALUES ('uuid-ของผู้ใช้', 'ซื้อของที่ตลาด');

แก้ไขข้อมูล

sql⚡ SQL Editor
UPDATE tasks
SET completed = true
WHERE id = 'uuid-ของ-task';

ลบข้อมูล

sql⚡ SQL Editor
-- ลบแถวที่ระบุ
DELETE FROM tasks WHERE id = 'uuid-ของ-task';
🚨 อันตราย! DELETE และ DROP

DELETE FROM tasks; (ไม่มี WHERE) จะลบข้อมูลทั้งตาราง!
DROP TABLE tasks; จะลบตารางทั้งตัว!
ตรวจสอบ WHERE ให้แน่ใจทุกครั้งก่อนรัน DELETE! และห้ามรัน DROP บน Production โดยไม่มี Backup!

⚡ Realtime — อัปเดตข้อมูลทันที

Realtime ทำให้เว็บอัปเดตข้อมูลทันทีที่มีการเปลี่ยนแปลง ไม่ต้อง refresh!

ts📄 ตัวอย่างโค้ด — ฟังการเปลี่ยนแปลง tasks
// ฟังเมื่อมีการเพิ่ม/แก้/ลบ Task
const channel = supabase
  .channel('tasks-changes')
  .on(
    'postgres_changes',
    {
      event: '*',          // ฟังทุก event (INSERT/UPDATE/DELETE)
      schema: 'public',
      table: 'tasks'
    },
    (payload) => {
      console.log('ข้อมูลเปลี่ยน:', payload)
      // โหลด Tasks ใหม่
      loadTasks()
    }
  )
  .subscribe()

// ยกเลิกการฟังเมื่อไม่ใช้แล้ว
// supabase.removeChannel(channel)
💡 ใช้ Realtime เมื่อไหร่?

เหมาะกับ: แชท, แจ้งเตือน, Dashboard สด, collaborative editing — ไม่ต้องใช้ทุกโปรเจกต์ ถ้าข้อมูลไม่ต้องอัปเดตทันทีก็ไม่ต้องใช้
