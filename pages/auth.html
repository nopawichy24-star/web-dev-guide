Authentication (ระบบสมาชิก)

🟢 Beginner ⏱️ อ่าน ~12 นาที
📋 Prerequisite
✓ Supabase พื้นฐาน
✓ JavaScript

Supabase มีระบบ Authentication ในตัว รองรับ Email/Password, Magic Link, Google, LINE และอื่นๆ

🔄 Auth Flow ตั้งแต่สมัครจนเข้าสู่ระบบ

📝 ผู้ใช้กรอกอีเมล + รหัสผ่าน แล้วกด "สมัคร"
⬇️
📧 Supabase ส่งอีเมลยืนยัน
⬇️
✅ ผู้ใช้คลิกลิงก์ในอีเมล → กลับมาที่เว็บ
⬇️
🔑 Supabase สร้าง Session
⬇️
🏠 ผู้ใช้เข้าสู่หน้า Dashboard ได้

📝 ตัวอย่างโค้ด

สมัครสมาชิก

ts📄 ตัวอย่างโค้ด
const { data, error } = await supabase.auth.signUp({
  email: 'user@example.com',
  password: 'securepassword123',
})

if (error) {
  alert('สมัครไม่สำเร็จ: ' + error.message)
} else {
  alert('สมัครสำเร็จ! กรุณาตรวจสอบอีเมลเพื่อยืนยัน')
}

เข้าสู่ระบบ

ts📄 ตัวอย่างโค้ด
const { data, error } = await supabase.auth.signInWithPassword({
  email: 'user@example.com',
  password: 'securepassword123',
})

if (error) {
  alert('เข้าสู่ระบบไม่สำเร็จ: ' + error.message)
} else {
  // เข้าสู่ระบบสำเร็จ! data.user มีข้อมูลผู้ใช้
  console.log('ยินดีต้อนรับ:', data.user.email)
}

ออกจากระบบ

ts📄 ตัวอย่างโค้ด
const { error } = await supabase.auth.signOut()
if (!error) {
  // redirect ไปหน้า login
  window.location.href = '/login'
}

ตรวจสอบ Session / ดึงข้อมูล User

ts📄 ตัวอย่างโค้ด
// ดึง User ปัจจุบัน
const { data: { user } } = await supabase.auth.getUser()

if (user) {
  console.log('เข้าสู่ระบบอยู่:', user.email)
  console.log('User ID:', user.id)    // ใช้กับ RLS!
} else {
  console.log('ยังไม่ได้เข้าสู่ระบบ')
  // redirect ไปหน้า login
}

Reset Password

ts📄 ตัวอย่างโค้ด
const { error } = await supabase.auth.resetPasswordForEmail(
  'user@example.com',
  { redirectTo: 'https://mysite.com/reset-password' }
)

if (!error) {
  alert('ส่งลิงก์รีเซ็ตรหัสผ่านไปที่อีเมลแล้ว!')
}

🔵 Login ด้วย Google (OAuth)

ตั้งค่าใน Supabase Dashboard ก่อน: Authentication → Providers → Google → เปิด → ใส่ Client ID + Client Secret จาก Google Cloud Console

ts📄 ตัวอย่างโค้ด
async function signInWithGoogle() {
  const { error } = await supabase.auth.signInWithOAuth({
    provider: 'google',
    options: {
      redirectTo: window.location.origin + '/tasks'
    }
  })
  if (error) alert('Login ไม่สำเร็จ: ' + error.message)
  // ผู้ใช้จะถูก redirect ไป Google แล้วกลับมาอัตโนมัติ
}

👂 ฟัง Session เปลี่ยนแปลง (onAuthStateChange)

ts📄 ใช้ใน layout หรือ component หลัก
import { useEffect } from 'react'
import { supabase } from '@/lib/supabase'
import { useRouter } from 'next/navigation'

function useAuth() {
  const router = useRouter()

  useEffect(() => {
    // ฟังทุกครั้งที่ Login/Logout
    const { data: { subscription } } = supabase.auth.onAuthStateChange(
      (event, session) => {
        if (event === 'SIGNED_IN') {
          router.push('/tasks')
        }
        if (event === 'SIGNED_OUT') {
          router.push('/login')
        }
      }
    )

    // cleanup เมื่อ component ถูกทำลาย
    return () => subscription.unsubscribe()
  }, [])
}
💜 ทำไมต้องใช้ onAuthStateChange?

เพราะ Session อาจเปลี่ยนได้ตลอดเวลา — Token หมดอายุ, ผู้ใช้ Logout จากแท็บอื่น, OAuth redirect กลับมา ฟังก์ชันนี้จะจับทุกเหตุการณ์อัตโนมัติ

⚙️ ตั้ง Redirect URL (สำคัญมาก!)

Supabase ต้องรู้ว่าหลัง Login/สมัคร ให้ส่งผู้ใช้กลับไปที่ URL ไหน:

  1. Supabase Dashboard → Authentication → URL Configuration
  2. ตั้ง Site URL: https://your-site.vercel.app
  3. เพิ่ม Redirect URLs:
    text
    http://localhost:3000/**
https://your-site.vercel.app/**
🚨 ปัญหาที่พบบ่อยที่สุด!

ถ้าลืมเพิ่ม Redirect URL → Login สำเร็จแต่ไม่กลับมาที่เว็บ! ต้องใส่ทั้ง localhost (สำหรับ dev) และ URL จริง (สำหรับ production)

🔧 ปัญหาที่พบบ่อย

ปัญหาสาเหตุวิธีแก้
อีเมลยืนยันไม่มาตรวจ Spam folder หรือ Supabase ส่งช้าดู Spam, รอ 1-2 นาที, ตรวจ SMTP settings
Redirect URL ผิดลืมเพิ่ม URL ใน Supabase Dashboardเพิ่มทั้ง localhost + production URL
Login แล้ว Session หายปิดเบราว์เซอร์แล้ว Token หมดอายุใช้ onAuthStateChange ฟัง Session
localhost ได้แต่ Production ไม่ได้Redirect URL หรือ Env Vars ไม่ตรงตรวจ URL Config + Env Vars บน Vercel
