คู่มือ Vercel ฉบับละเอียด

🟢 Beginner ⏱️ อ่าน ~12 นาที
📋 Prerequisite
✓ GitHub พื้นฐาน
✓ มี Vercel Account

Vercel คือแพลตฟอร์มที่ช่วยนำเว็บไซต์ของเราขึ้นออนไลน์ เชื่อมกับ GitHub แล้วทุกครั้งที่เรา Push โค้ด เว็บจะอัปเดตอัตโนมัติ

🤔 Vercel เหมาะกับโปรเจกต์แบบใด?

ประเภทคำอธิบายตัวอย่าง
Static Websiteเว็บไซต์ HTML/CSS/JS ธรรมดาPortfolio, Landing page
SPASingle Page ApplicationReact app, Vue app
Next.js⭐ รองรับดีที่สุด (Vercel สร้าง Next.js เอง!)เว็บแอปที่มี Backend
Serverless FunctionsAPI ที่รันเมื่อถูกเรียกเท่านั้นAPI endpoint, Webhook

📝 สมัคร Vercel

  1. เข้าเว็บ vercel.com
  2. กด Sign Up
  3. เลือก Continue with GitHub (แนะนำ! เชื่อมทีเดียวจบ)
  4. อนุญาตให้ Vercel เข้าถึง GitHub
  5. เลือกแผน Hobby (ฟรี)

🚀 Deploy เว็บไซต์ครั้งแรก (ทีละขั้นตอน)

ขั้นตอนที่ 1: Import Repository

  1. เข้า Vercel Dashboard → กด Add New... → Project
  2. จะเห็นรายชื่อ Repo จาก GitHub → กด Import ที่ Repo ที่ต้องการ
  3. ถ้าไม่เห็น Repo → กด Adjust GitHub App Permissions

ขั้นตอนที่ 2: ตั้งค่า

ตั้งค่าคำอธิบายตัวอย่าง
Framework Presetเลือก Framework (ปกติ Vercel ตรวจจับเอง)Next.js, React, Other
Root Directoryโฟลเดอร์ที่โค้ดอยู่./ (root)
Build Commandคำสั่งสร้างเว็บnpm run build
Output Directoryโฟลเดอร์ผลลัพธ์.next (Next.js), dist

ขั้นตอนที่ 3: เพิ่ม Environment Variables

กด Environment Variables แล้วเพิ่ม:

env⚙️ Vercel Dashboard → Environment Variables
NEXT_PUBLIC_SUPABASE_URL = https://xxxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY = eyJhbGc...
SUPABASE_SERVICE_ROLE_KEY = eyJhbGc...
🚨 ห้ามใส่ Service Role Key ในโค้ด!

ตัวแปรที่ขึ้นต้นด้วย NEXT_PUBLIC_ จะถูกเปิดเผยใน Browser! ดังนั้น SUPABASE_SERVICE_ROLE_KEY ต้องไม่มี prefix NEXT_PUBLIC_

ขั้นตอนที่ 4: กด Deploy!

กดปุ่ม Deploy → Vercel จะ Build โค้ดของเรา → ถ้าสำเร็จจะได้ URL เว็บไซต์ เช่น my-project.vercel.app

🌍 3 สภาพแวดล้อม ของ Vercel

สภาพแวดล้อมคืออะไรเมื่อไหร่
💻 Development เครื่องเราที่ localhost:3000 ตอนเขียนโค้ด
👁️ Preview URL ชั่วคราวทุกครั้งที่ Push branch ที่ไม่ใช่ main ทดสอบก่อน merge
🌍 Production เว็บไซต์จริงที่ your-project.vercel.app เมื่อ Push ไป main
💜 ประโยชน์ของ Preview

ทุกครั้งที่เราสร้าง Pull Request Vercel จะสร้าง URL ตัวอย่างให้ทดสอบก่อน ถ้าใช้ได้ดีค่อย Merge เข้า main → Production อัปเดตอัตโนมัติ!

🔧 ปัญหาที่พบบ่อยบน Vercel

ปัญหาสาเหตุที่เป็นไปได้วิธีแก้
❌ Build failed โค้ดมี Error, import ผิด, TypeScript error อ่าน Build Logs ใน Vercel → แก้ Error ที่ระบุ → Push ใหม่
❌ Module not found ลืมติดตั้ง package หรือ import path ผิด รัน npm install แล้ว Push ใหม่ ตรวจ case ชื่อไฟล์
❌ Env variable undefined ลืมเพิ่มใน Vercel Dashboard หรือชื่อผิด ตรวจชื่อตัวแปรให้ตรงกับโค้ด ตรวจ prefix NEXT_PUBLIC_
❌ Deploy สำเร็จแต่หน้าขาว Root Directory ผิด หรือ Framework ถูกตรวจจับผิด ไปที่ Settings → General → ตรวจ Framework Preset + Root Directory
❌ Route เปิดตรงแล้ว 404 SPA ต้อง config rewrites Next.js ไม่มีปัญหานี้ / SPA อื่นต้องเพิ่ม vercel.json rewrites
❌ CORS error เรียก API จากโดเมนอื่นโดยไม่ได้ตั้ง headers เพิ่ม CORS headers ในไฟล์ vercel.json หรือ API route
❌ Case-sensitive filename Mac/Windows ไม่แยกตัวพิมพ์ แต่ Vercel (Linux) แยก Header.tsx ≠ header.tsx → ตรวจชื่อไฟล์ให้ตรง import
💡 วิธีดู Build Logs

Vercel Dashboard → เลือกโปรเจกต์ → Deployments → กดที่ Deployment ล่าสุด → ดู Build Logs → หา Error สีแดง — มันจะบอกว่าบรรทัดไหนมีปัญหา

🔄 Rollback — ย้อนกลับเมื่อเว็บพัง

ถ้า Deploy แล้วเว็บมีปัญหา สามารถกลับไปเวอร์ชันก่อนหน้าได้:

  1. เข้า Vercel Dashboard → Deployments
  2. หา Deployment ก่อนหน้าที่ใช้งานได้ดี
  3. กดจุด 3 จุด (⋯) → Promote to Production
✅ เว็บกลับมาใช้ได้ทันที!

แล้วค่อยแก้โค้ดในเครื่อง → Push ใหม่ → Vercel จะ Deploy เวอร์ชันที่แก้แล้วอัตโนมัติ

🌐 เชื่อม Custom Domain

  1. Vercel Dashboard → Settings → Domains
  2. พิมพ์โดเมนของคุณ เช่น mysite.com
  3. Vercel จะบอกค่า DNS ที่ต้องตั้ง
  4. ไปตั้งค่า DNS ที่เว็บผู้ให้บริการโดเมน (เช่น Namecheap, Cloudflare)
  5. รอ DNS อัปเดต (อาจใช้เวลา 1-48 ชม.)
  6. Vercel จะสร้าง SSL/HTTPS ให้อัตโนมัติ! 🔒
