1️⃣ Component — ชิ้นส่วนของหน้าเว็บ

ทุกอย่างใน React คือ Component — ปุ่ม, ฟอร์ม, เมนู, การ์ด ล้วนเป็น Component ทั้งนั้น

🌐 หน้าเว็บทั้งหน้า
ประกอบจาก Component หลายตัว
⬇️
📦 Header + Sidebar + Content + Footer
แต่ละส่วนเป็น Component แยก
⬇️
🧱 Button, Card, Input, Modal...
Component เล็กสุดที่ใช้ซ้ำได้
tsx📄 components/Button.tsx — Component แรกของเรา!
// Component คือ "ฟังก์ชันที่ return HTML"
function Button() {
  return (
    <button className="bg-blue-500 text-white px-4 py-2 rounded">
      กดฉัน!
    </button>
  )
}

export default Button
💜 กฎ Component

1) ชื่อต้องขึ้นต้นด้วยตัวพิมพ์ใหญ่ เช่น Button ไม่ใช่ button
2) ต้อง return HTML (จริงๆ คือ JSX)
3) สร้างครั้งเดียว ใช้ซ้ำได้ทุกที่: <Button />

3️⃣ Props — ส่งข้อมูลให้ Component

Props คือข้อมูลที่ Component แม่ส่งให้ Component ลูก — เหมือนพ่อแม่ส่งจดหมายให้ลูก

tsx📄 ตัวอย่าง Props
// Component ลูก — รับ Props
function UserCard({ name, age }: { name: string; age: number }) {
  return (
    <div className="p-4 border rounded">
      <h2>{name}</h2>
      <p>อายุ {age} ปี</p>
    </div>
  )
}

// Component แม่ — ส่ง Props
function App() {
  return (
    <div>
      <UserCard name="สมชาย" age={25} />
      <UserCard name="สมศรี" age={30} />
      {/* Component เดียวกัน แต่ข้อมูลต่างกัน! */}
    </div>
  )
}
⚠️ Props เป็น Read-Only!

Component ลูกห้ามแก้ไข Props ที่ได้มา — ถ้าต้องการข้อมูลที่เปลี่ยนได้ ต้องใช้ State

4️⃣ Event — ตอบสนองต่อผู้ใช้

tsx📄 ตัวอย่าง Event
function EventDemo() {
  // ฟังก์ชันที่ทำงานเมื่อเกิด Event
  function handleClick() {
    alert('คลิกแล้ว!')
  }

  function handleChange(e: React.ChangeEvent<HTMLInputElement>) {
    console.log('พิมพ์:', e.target.value)
  }

  function handleSubmit(e: React.FormEvent) {
    e.preventDefault()    // ป้องกันหน้า reload
    alert('ส่งฟอร์มแล้ว!')
  }

  return (
    <div>
      <button onClick={handleClick}>กดฉัน</button>
      <input onChange={handleChange} placeholder="พิมพ์อะไรสักอย่าง" />
      <form onSubmit={handleSubmit}>
        <button type="submit">ส่ง</button>
      </form>
    </div>
  )
}
Eventเมื่อไหร่ใช้กับ
onClickเมื่อคลิกปุ่ม, ลิงก์, div
onChangeเมื่อค่าเปลี่ยนinput, select, textarea
onSubmitเมื่อส่งฟอร์มform
onKeyDownเมื่อกดคีย์บอร์ดinput, div

6️⃣ useState — Hook สำคัญที่สุด ⭐

tsx📄 ตัวอย่าง useState — Counter
import { useState } from 'react'

function Counter() {
  //        ค่า      ฟังก์ชันเปลี่ยนค่า    ค่าเริ่มต้น
  const [count, setCount] = useState(0)

  return (
    <div>
      <p>กดแล้ว {count} ครั้ง</p>
      <button onClick={() => setCount(count + 1)}>+1</button>
      <button onClick={() => setCount(count - 1)}>-1</button>
      <button onClick={() => setCount(0)}>Reset</button>
    </div>
  )
}
tsx📄 ตัวอย่าง useState — ฟอร์ม Input
import { useState } from 'react'

function SearchBox() {
  const [query, setQuery] = useState('')       // เก็บข้อความค้นหา
  const [results, setResults] = useState([])   // เก็บผลลัพธ์

  return (
    <div>
      <input
        value={query}
        onChange={(e) => setQuery(e.target.value)}
        placeholder="ค้นหา..."
      />
      <p>กำลังค้นหา: {query}</p>
    </div>
  )
}
🚨 กฎ useState

❌ ห้ามแก้ State ตรงๆ: count = 5 ← ไม่ทำงาน!
✅ ต้องใช้ setter: setCount(5) ← React ถึงจะรู้ว่าต้อง re-render

❌ ห้ามเรียก useState ใน if/loop — ต้องอยู่บนสุดของ Component เสมอ

Hook คือฟังก์ชันที่ขึ้นต้นด้วย use — ใช้เพิ่มความสามารถให้ Component

Hookใช้ทำอะไรใช้บ่อยแค่ไหน

8️⃣ Rendering — React วาดหน้าจออย่างไร

Conditional Rendering

tsx📄 แสดง Component ตามเงื่อนไข
function Dashboard({ user }: { user: any | null }) {
  // ถ้ายังไม่ login → แสดงหน้า Login
  if (!user) {
    return <LoginPage />
  }

  // ถ้า login แล้ว → แสดง Dashboard
  return (
    <div>
      <h1>ยินดีต้อนรับ {user.name}</h1>
      {user.isAdmin && <AdminPanel />}  {/* แสดงเฉพาะ Admin */}
    </div>
  )
}

Render รายการ (List)

tsx📄 ใช้ .map() สร้างรายการ
function TaskList({ tasks }: { tasks: { id: string; title: string }[] }) {
  if (tasks.length === 0) {
    return <p>📭 ยังไม่มี Task</p>   // Empty state
  }

  return (
    <ul>
      {tasks.map(task => (
        <li key={task.id}>{task.title}</li>
        {/* ⚠️ ต้องมี key ที่ไม่ซ้ำกัน! */}
      ))}
    </ul>
  )
}
⚠️ ต้องมี key ใน List!

เมื่อใช้ .map() สร้างรายการ ทุก element ต้องมี key ที่ไม่ซ้ำกัน — React ใช้ key บอกว่าแถวไหนเปลี่ยน ห้ามใช้ index เป็น key!

9️⃣ useEffect — ทำงานหลัง Render ⭐

ใช้ทำงานที่ต้องเกิดหลังจากหน้าจอแสดงผลแล้ว เช่น โหลดข้อมูลจาก Supabase

tsx📄 ตัวอย่าง useEffect — โหลดข้อมูลตอนเปิดหน้า
import { useState, useEffect } from 'react'
import { supabase } from '@/lib/supabase'

function TaskPage() {
  const [tasks, setTasks] = useState([])
  const [loading, setLoading] = useState(true)

  // useEffect รันหลังจาก Component แสดงผลครั้งแรก
  useEffect(() => {
    async function loadTasks() {
      const { data } = await supabase.from('tasks').select('*')
      if (data) setTasks(data)
      setLoading(false)
    }
    loadTasks()
  }, [])  // ← [] = รันครั้งเดียวตอนเปิดหน้า

  if (loading) return <p>⏳ กำลังโหลด...</p>

  return (
    <ul>
      {tasks.map(t => <li key={t.id}>{t.title}</li>)}
    </ul>
  )
}
Dependency Arrayรันเมื่อตัวอย่าง
[]ครั้งเดียว ตอนเปิดหน้าโหลดข้อมูลครั้งแรก
[count]ทุกครั้งที่ count เปลี่ยนอัปเดตเมื่อค่าเปลี่ยน
ไม่ใส่ทุกครั้งที่ re-render ⚠️ระวัง! อาจทำงานซ้ำไม่หยุด
❌ Common Mistake — Infinite Loop!

ถ้าลืมใส่ [] ใน useEffect + มี setState ข้างใน → จะวน loop ไม่หยุด! เพราะ setState → re-render → useEffect รันใหม่ → setState → re-render → ...

🔟 Context — แชร์ข้อมูลทั้งแอป

ปกติต้องส่ง Props จากแม่ → ลูก → หลาน ทีละชั้น (เรียก Prop Drilling) แต่ Context ให้ทุก Component เข้าถึงข้อมูลได้เลย!

tsx📄 ตัวอย่าง Context — แชร์ข้อมูล User ทั้งแอป
import { createContext, useContext, useState } from 'react'

// 1. สร้าง Context
const UserContext = createContext(null)

// 2. สร้าง Provider (ครอบทั้งแอป)
function UserProvider({ children }) {
  const [user, setUser] = useState(null)
  return (
    <UserContext.Provider value={{ user, setUser }}>
      {children}
    </UserContext.Provider>
  )
}

// 3. ใช้ใน Component ไหนก็ได้!
function Navbar() {
  const { user } = useContext(UserContext)
  return <p>สวัสดี {user?.name || 'Guest'}</p>
}

function LoginButton() {
  const { setUser } = useContext(UserContext)
  return (
    <button onClick={() => setUser({ name: 'สมชาย' })}>
      Login
    </button>
  )
}
💜 ใช้ Context เมื่อไหร่?

✅ ข้อมูลที่ใช้ทั่วแอป: User, Theme (Dark/Light), ภาษา
❌ ข้อมูลเฉพาะ Component: ค่า input, สถานะ loading → ใช้ useState ธรรมดา
