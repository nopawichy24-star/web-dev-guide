ก่อนจะสร้างเว็บได้ เราต้องเข้าใจก่อนว่าเว็บไซต์ทำงานอย่างไร และมีส่วนประกอบอะไรบ้าง

🌍 เว็บไซต์ทำงานอย่างไร?

เมื่อเราพิมพ์ที่อยู่เว็บไซต์ (เช่น google.com) ในเบราว์เซอร์แล้วกด Enter สิ่งที่เกิดขึ้นคือ:

🖥️ เบราว์เซอร์ส่งคำขอไปยัง Server
⬇️
🔍 DNS แปลงชื่อเว็บเป็นที่อยู่ IP
⬇️
📦 Server ส่งไฟล์เว็บ (HTML, CSS, JavaScript) กลับมา
⬇️
🎨 เบราว์เซอร์แสดงผลหน้าเว็บให้เราเห็น

🏗️ ส่วนประกอบของเว็บไซต์

1. Frontend (ส่วนหน้าบ้าน)

คือทุกสิ่งที่ผู้ใช้เห็นและโต้ตอบได้บนหน้าจอ ประกอบด้วย 3 ภาษาหลัก:

ภาษาหน้าที่เปรียบเทียบ
HTML โครงสร้างเนื้อหา — กำหนดว่ามีอะไรบ้าง 🦴 โครงกระดูกของร่างกาย
CSS รูปลักษณ์ — กำหนดหน้าตา สี ขนาด 👔 เสื้อผ้าและแต่งหน้า
JavaScript พฤติกรรม — ทำให้เว็บตอบสนอง 🧠 สมองและกล้ามเนื้อ

ตัวอย่าง HTML ง่ายๆ:

HTML 📄 ไฟล์ index.html 
<!DOCTYPE html>
<html lang="th">
<head>
  <title>เว็บไซต์แรกของฉัน</title>
</head>
<body>
  <h1>สวัสดี!</h1>
  <p>นี่คือเว็บไซต์แรกของฉัน</p>
  <button onclick="alert('คลิกแล้ว!')">กดฉัน</button>
</body>
</html>

2. Backend (ส่วนหลังบ้าน)

คือส่วนที่ทำงานเบื้องหลัง ผู้ใช้ไม่เห็นโดยตรง ทำหน้าที่:

  • รับข้อมูลจากผู้ใช้ (เช่น ฟอร์มสมัครสมาชิก)
  • ตรวจสอบสิทธิ์ (เช่น รหัสผ่านถูกไหม?)
  • อ่านและเขียนข้อมูลในฐานข้อมูล
  • ส่งอีเมล, ประมวลผลรูปภาพ ฯลฯ
💡 สำหรับคู่มือนี้

เราจะใช้ Supabase เป็น Backend ไม่ต้องเขียน Backend เอง — ช่วยให้ผู้เริ่มต้นสร้างเว็บแอปได้เร็วขึ้นมาก!

3. Database (ฐานข้อมูล)

คือที่เก็บข้อมูลทั้งหมดของเว็บไซต์ เช่น:

  • ข้อมูลสมาชิก (ชื่อ, อีเมล, รหัสผ่าน)
  • บทความ, โพสต์, ความคิดเห็น
  • รายการสินค้า, คำสั่งซื้อ

ลองนึกภาพว่า Database เหมือนตาราง Excel ขนาดใหญ่:

idusernameemailcreated_at

2somsrisomsri@email.com2026-02-20

4. Hosting (ที่พักเว็บไซต์)

เว็บไซต์ต้อง "อาศัยอยู่" ที่ไหนสักที่บนอินเทอร์เน็ต เพื่อให้คนอื่นเข้าถึงได้

ในคู่มือนี้เราใช้ Vercel เป็น Hosting ซึ่งจะ auto-deploy ทุกครั้งที่เราส่งโค้ดไปที่ GitHub

🔗 ทั้ง 4 ส่วนเชื่อมกันอย่างไร?

🎨 Frontend

ผู้ใช้กดปุ่ม "สมัครสมาชิก" → ส่งข้อมูลไป Backend

⚙️ Backend

รับข้อมูล → ตรวจสอบ → บันทึกลง Database

🗄️ Database

เก็บข้อมูลผู้ใช้ใหม่ → พร้อมให้เรียกใช้ทีหลัง

🏨 Hosting

ทำให้ทั้งหมดเข้าถึงได้ผ่านอินเทอร์เน็ต 24 ชม.

🆚 Static Website vs Dynamic Website

📄 Static Website

เนื้อหาเหมือนกันทุกคน ไม่มีระบบสมาชิก ไม่มีฐานข้อมูล

ตัวอย่าง: หน้าเว็บประวัติส่วนตัว, บล็อกส่วนตัว

⚡ Dynamic Website

เนื้อหาเปลี่ยนตามผู้ใช้ มีระบบ Login มีฐานข้อมูล

ตัวอย่าง: Facebook, Twitter, ร้านค้าออนไลน์

✅ สิ่งที่ควรเข้าใจแล้วตอนนี้

เว็บไซต์ = Frontend (หน้าตา) + Backend (เบื้องหลัง) + Database (ข้อมูล) + Hosting (ที่พัก)

🔗 API คืออะไร?

API คือ "พนักงานเสิร์ฟ" ระหว่าง Frontend กับ Database:

🖥️ Frontend (ลูกค้า)
"ขอรายการ Task ทั้งหมด"
⬇️ ส่ง Request
🔗 API (พนักงานเสิร์ฟ)
รับคำสั่ง → ตรวจสิทธิ์ → ดึงข้อมูล
⬇️ ⬆️
🗄️ Database (ห้องครัว)
เก็บข้อมูลจริง → ส่งกลับ
⬆️ ส่ง Response
🖥️ Frontend แสดงผลให้ผู้ใช้เห็น

📡 HTTP Request — วิธีที่ Browser คุยกับ Server

เมื่อเว็บต้องการข้อมูลจาก Server จะส่ง HTTP Request มี 4 แบบหลัก:

Methodความหมายใช้เมื่อSupabase
🟢 GETขอข้อมูล (อ่าน)โหลดรายการ Task.select()
🔵 POSTส่งข้อมูลใหม่ (สร้าง)สร้าง Task ใหม่.insert()
🟡 PUT / PATCHแก้ไขข้อมูลเปลี่ยนชื่อ Task.update()
🔴 DELETEลบข้อมูลลบ Task.delete()
💡 ข่าวดี!

Supabase JS Library จัดการ HTTP Request ให้เราอัตโนมัติ — เราแค่เขียน .select() หรือ .insert() ไม่ต้องเขียน HTTP เอง!

⚛️ React เบื้องต้น — Render, State, Re-render

React ทำงาน 3 จังหวะ:

1️⃣ Render
แสดงหน้าเว็บครั้งแรก
⬇️
2️⃣ State เปลี่ยน
เช่น ผู้ใช้กดปุ่ม, พิมพ์ข้อความ, โหลดข้อมูลเสร็จ
⬇️
3️⃣ Re-render
React อัปเดตหน้าเว็บอัตโนมัติ!
tsx📄 ตัวอย่าง State + Re-render
import { useState } from 'react'

function Counter() {
  // count = ค่าปัจจุบัน, setCount = ฟังก์ชันเปลี่ยนค่า
  const [count, setCount] = useState(0)

  return (
    <div>
      <p>กดแล้ว {count} ครั้ง</p>
      <button onClick={() => setCount(count + 1)}>
        กดฉัน!
      </button>
    </div>
  )
  // เมื่อ setCount ถูกเรียก → count เปลี่ยน → React re-render → หน้าจออัปเดต!
}
