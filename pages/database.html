บทนี้เจาะลึกเรื่อง Relationship และ JOIN ซึ่งเป็นหัวใจของฐานข้อมูลเชิงสัมพันธ์

🔗 Relationship (ความสัมพันธ์ระหว่างตาราง)

ตารางสามารถเชื่อมกันได้ด้วย Foreign Key:

diagram
profiles               tasks
┌──────────┐          ┌──────────────┐
│ id (PK)  │◄─────────│ user_id (FK) │
│ username │          │ id (PK)      │
│ full_name│          │ title        │
└──────────┘          │ completed    │
                      └──────────────┘
1 ผู้ใช้ มีได้หลาย Tasks (One-to-Many)

📝 SQL ขั้นสูง

JOIN — รวมข้อมูลจากหลายตาราง

sql⚡ SQL Editor
-- แสดง Task พร้อมชื่อผู้ใช้
SELECT tasks.title, tasks.completed, profiles.username
FROM tasks
JOIN profiles ON tasks.user_id = profiles.id
ORDER BY tasks.created_at DESC;

นับจำนวน

sql⚡ SQL Editor
-- นับ Task ทั้งหมดของผู้ใช้แต่ละคน
SELECT profiles.username, COUNT(tasks.id) AS total_tasks
FROM profiles
LEFT JOIN tasks ON profiles.id = tasks.user_id
GROUP BY profiles.username;

แก้ไขโครงสร้างตาราง

sql⚡ SQL Editor
-- เพิ่ม Column ใหม่
ALTER TABLE tasks ADD COLUMN description TEXT;

-- เปลี่ยนชื่อ Column
ALTER TABLE tasks RENAME COLUMN title TO task_name;
⚠️ ระวังการแก้ไข Schema บน Production!

ทดสอบใน Development ก่อนเสมอ การลบ Column หรือ DROP TABLE บน Production ทำให้ข้อมูลหายถาวร ต้องมี Backup ก่อนทุกครั้ง!

⚡ เรียกข้อมูล Relationship จาก JavaScript

ts📄 ตัวอย่างโค้ด
// ดึง Task พร้อมข้อมูล Profile (เหมือน JOIN)
const { data } = await supabase
  .from('tasks')
  .select('*, profiles(username, full_name)')
  .order('created_at', { ascending: false })
