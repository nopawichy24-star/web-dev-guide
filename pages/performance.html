🤔 Performance คืออะไร?

คือความเร็วและประสิทธิภาพของเว็บ — ผู้ใช้จะออกจากเว็บถ้าโหลดนานเกิน 3 วินาที!

📊 Core Web Vitals

ตัวชี้วัดคืออะไรค่าที่ดี
LCPเว็บโหลดเสร็จเร็วแค่ไหน< 2.5 วินาที
INPกดแล้วตอบสนองเร็วแค่ไหน< 200ms
CLSหน้าเว็บกระโดดไหม< 0.1

🚨 ถ้าเว็บเริ่มช้า — ตรวจ 5 อย่างนี้ก่อน!

#ปัญหาวิธีแก้
1️⃣🖼️ รูปภาพใหญ่เกินไปใช้ next/image + format WebP + ลดขนาด
2️⃣📊 โหลดข้อมูลเยอะเกินใช้ Pagination หรือ .limit(20)
3️⃣📦 JavaScript ใหญ่เกินลบ package ที่ไม่ใช้ + ใช้ Lazy Loading
4️⃣🔄 ไม่มี Cacheตั้ง Cache headers ใน Vercel
5️⃣🗄️ Query ช้าเพิ่ม Index ใน Database

🖼️ Image Optimization ใน Next.js

tsx📄 ใช้ next/image แทน <img>
import Image from 'next/image'

// ✅ ดี — Next.js จะ optimize อัตโนมัติ
<Image src="/photo.jpg" alt="รูป" width={400} height={300} />

// ❌ ไม่ดี — ไม่ optimize
<img src="/photo.jpg" alt="รูป" />

📊 Pagination กับ Supabase

ts📄 โหลดข้อมูลทีละหน้า
const PAGE_SIZE = 20

async function loadPage(page: number) {
  const from = page * PAGE_SIZE
  const to = from + PAGE_SIZE - 1

  const { data } = await supabase
    .from('tasks')
    .select('*')
    .order('created_at', { ascending: false })
    .range(from, to)  // ดึงเฉพาะ 20 รายการต่อหน้า

  return data
}
